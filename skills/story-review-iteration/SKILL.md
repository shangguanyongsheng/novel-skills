# 短故事多轮审校迭代

> 输入一个短故事文件，自动完成多轮审校→修复→评估→迭代，直到分数收敛。

---

## 🎯 触发条件

用户说以下内容时触发：

- "审校这个故事"
- "检查/完善这个短故事"
- "多轮迭代修复"
- "帮我精修小说"
- "参考 Karpathy autoresearch 的思路审校"

## 📋 核心流程

```
┌──────────────────────────────────────────────────────────────────┐
│                    多轮审校迭代流程                                │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Step 0: 备份原稿                                                │
│  ├── cp story.md story.md.bak                                    │
│  └── 原稿永远不动，所有修改在新版本文件上                          │
│                                                                  │
│  Step 1: 首轮全面审校 (Pass 1-6)                                │
│  ├── Pass 1: 结构扫描 — 段落长度/内容缺失/顺序                   │
│  ├── Pass 2: P0 问题 — 内容缺失、矛盾、重复                      │
│  ├── Pass 3: P1 问题 — 比喻过多、词频过高、冗余                  │
│  ├── Pass 4: P2 问题 — 格式、标点、视角一致性                    │
│  ├── Pass 5: 历史/事实核查 — 时间线、数字、地名                  │
│  └── Pass 6: 平台适配度 — 开头钩子、高潮密度、字数                │
│                                                                  │
│  Step 2: 生成审校报告                                            │
│  ├── 按 P0/P1/P2 列出问题清单                                    │
│  ├── 标注具体位置、问题描述、修复建议                             │
│  └── 给出初始评分 (6 维度 × 10 分)                               │
│                                                                  │
│  Step 3: 循环修复 (Round N)                                     │
│  ├── 读取当前版本                                                 │
│  ├── 逐项修复 P0 问题 → 保存 V(n+1)                              │
│  ├── 逐项修复 P1 问题 → 保存 V(n+1)                              │
│  ├── 逐项修复 P2 问题 → 保存 V(n+1)                              │
│  └── 执行验证 (见下方验证清单)                                    │
│                                                                  │
│  Step 4: 迭代评估                                                │
│  ├── 对新版本重新评分                                             │
│  ├── 对比上一轮分数                                               │
│  └── 判断是否收敛                                                 │
│                                                                  │
│  Step 5: 收敛判断                                                │
│  ├── 条件 A: 评分 ≥ 90/100                                       │
│  ├── 条件 B: 连续两轮分数提升 < 3 分                             │
│  ├── 条件 C: 所有 P0 问题已清零                                  │
│  └── 满足任一条件 → 输出最终版，否则 → Step 3                    │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 🔴🟡🟢 问题分级定义

### P0 — 必须修复（影响阅读体验，扣分 ≥3）

| 类型 | 示例 |
|------|------|
| 内容缺失 | 某段落为空或过短（<100字） |
| 前后矛盾 | "三百里" vs "几百里" |
| 核心重复 | 同一段落/意象完整重复出现 |
| 视角跳变 | 说书人突然变成第一人称 |
| 情节断裂 | 关键事件没有铺垫或后续 |

### P1 — 建议修复（提升质量，扣分 1-3）

| 类型 | 阈值 | 修复方向 |
|------|------|----------|
| 比喻密度 | >40 个 | 精简到 40 以内，保留最有力的 |
| 句式重复 | >20 次同一句型 | 改为陈述句或换句型 |
| 关键词重复 | >6 次 | 用变体替代 |
| 结尾散漫 | 最后100字无力 | 收束到3-4个有力短句 |
| 历史细节 | 事实错误 | 修正为准确数字/事件 |

### P2 — 可选优化（锦上添花，加分 1-2）

| 类型 | 修复方向 |
|------|----------|
| 格式问题 | 引号配对、段落标记 |
| 视角一致性 | 统一叙事者视角 |
| 年龄/时间暗示 | 补充隐性信息 |
| 标点规范 | 统一中英文标点 |

## 📊 评分体系 (6 维度 × 10 分 = 60 满分)

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 结构完整性 | 10% | 段落长度均匀、无缺失、顺序合理 |
| 历史准确性 | 10% | 时间线、人物、事件、数字正确 |
| 张力与节奏 | 15% | 高潮密度、悬念设置、节奏变化 |
| 语言精炼度 | 20% | 比喻密度、词频、冗余度 |
| 平台适配度 | 15% | 开头钩子、高潮密度、字数达标 |
| 情感连贯性 | 30% | 人物情感线、意象呼应、伏笔回收 |

**换算**: 总分 / 60 × 100 = 百分制分数

## 📁 文件命名规范

| 文件 | 命名 | 说明 |
|------|------|------|
| 原稿 | `story.md` | 永远不动 |
| 备份 | `story.md.bak` | 首轮自动创建 |
| 中间版 | `story_v2.md`, `story_v3.md` | 每轮迭代生成 |
| 最终版 | `story_vN_final.md` | 收敛时输出 |
| 审校报告 | `review_report_round1.md` | 每轮一份 |

---

## ✅ 每轮迭代验证清单

每次修复后，必须执行以下验证：

```
1. 段落顺序完整性: 01 → 02 → ... → N (无跳段)
2. 段落长度均匀性: 最短段 ≥ 100 字，最长段 ≤ 1200 字
3. P0 问题清零验证: 逐个检查上一轮 P0 项
4. P1 阈值验证: 比喻≤40, "不是"≤20, 关键词≤6
5. 引号配对: 左引号数 = 右引号数
6. 重复短语检查: 同一短语连续重复 ≤ 2 处
7. 关键意象唯一性: 核心意象（如"抱不住"）只出现 1 次
8. 字数达标: 总字数 ≥ 平台最低要求（如番茄 ≥8000）
```

## 🔄 收敛条件（满足任一即停止）

| 条件 | 说明 |
|------|------|
| 评分 ≥ 90 | 达到优秀水平 |
| 连续两轮 Δ < 3 | 边际收益递减 |
| 所有 P0 清零 | 无必须修复问题 |
| 最大轮次 = 8 | 防止无限循环 |

## ⚠️ 注意事项

1. **原稿永远不动**: 所有修改在 `story_vN.md` 上
2. **每轮一个文件**: V2 → V3 → V4，不覆盖上一版本
3. **系统化处理**: 不要跳过任何 P0 问题
4. **验证先行**: 修复前先验证问题存在，修复后再验证已清除
5. **保留风格**: 短句重复等刻意写作手法不要误杀
6. **自动循环**: 不要等用户确认每一轮，自动执行直到收敛
7. **最终汇报**: 收敛后给出完整版本历史对比表
