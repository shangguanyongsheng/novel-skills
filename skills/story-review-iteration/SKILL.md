# 短故事多轮审校迭代

> 输入一个短故事文件，自动完成多轮审校→修复→评估→迭代，直到分数收敛。

## 📋 核心流程

```
┌──────────────────────────────────────────────────────────────────┐
│                    多轮审校迭代流程                                │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Step 0: 备份原稿                                                │
│  ├── cp story.md story.md.bak                                    │
│  └── 原稿永远不动，所有修改在新版本文件上                          │
│                                                                  │
│  Step 1: 首轮全面审校 (Pass 1-6)                                │
│  ├── Pass 1: 结构扫描 — 段落长度/内容缺失/顺序                   │
│  ├── Pass 2: P0 问题 — 内容缺失、矛盾、重复                      │
│  ├── Pass 3: P1 问题 — 比喻过多、词频过高、冗余                  │
│  ├── Pass 4: P2 问题 — 格式、标点、视角一致性                    │
│  ├── Pass 5: 历史/事实核查 — 时间线、数字、地名                  │
│  └── Pass 6: 平台适配度 — 开头钩子、高潮密度、字数                │
│                                                                  │
│  Step 2: 生成审校报告                                            │
│  ├── 按 P0/P1/P2 列出问题清单                                    │
│  ├── 标注具体位置、问题描述、修复建议                             │
│  └── 给出初始评分 (6 维度 × 10 分)                               │
│                                                                  │
│  Step 3: 循环修复 (Round N)                                     │
│  ├── 读取当前版本                                                 │
│  ├── 逐项修复 P0 问题 → 保存 V(n+1)                              │
│  ├── 逐项修复 P1 问题 → 保存 V(n+1)                              │
│  ├── 逐项修复 P2 问题 → 保存 V(n+1)                              │
│  └── 执行验证 (见下方验证清单)                                    │
│                                                                  │
│  Step 4: 迭代评估                                                │
│  ├── 对新版本重新评分                                             │
│  ├── 对比上一轮分数                                               │
│  └── 判断是否收敛                                                 │
│                                                                  │
│  Step 5: 收敛判断                                                │
│  ├── 条件 A: 评分 ≥ 90/100                                       │
│  ├── 条件 B: 连续两轮分数提升 < 3 分                             │
│  ├── 条件 C: 所有 P0 问题已清零                                  │
│  └── 满足任一条件 → 输出最终版，否则 → Step 3                    │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

## 🔴🟡🟢 问题分级定义

### P0 — 必须修复（影响阅读体验，扣分 ≥3）

| 类型 | 示例 |
|------|------|
| 内容缺失 | 某段落为空或过短（<100字） |
| 前后矛盾 | "三百里" vs "几百里" |
| 核心重复 | 同一段落/意象完整重复出现 |
| 视角跳变 | 说书人突然变成第一人称 |
| 情节断裂 | 关键事件没有铺垫或后续 |

### P1 — 建议修复（提升质量，扣分 1-3）

| 类型 | 阈值 | 修复方向 |
|------|------|----------|
| 比喻密度 | >40 个 | 精简到 40 以内，保留最有力的 |
| 句式重复 | >20 次同一句型 | 改为陈述句或换句型 |
| 关键词重复 | >6 次 | 用变体替代 |
| 结尾散漫 | 最后100字无力 | 收束到3-4个有力短句 |
| 历史细节 | 事实错误 | 修正为准确数字/事件 |
| **AI 味道** | >10 处 AI 痕迹词 | 替换为人类口语化表达 |
| **长句拆分** | >15 字句子占比 >30% | 拆成 2-3 个短句 |

### P2 — 可选优化（锦上添花，加分 1-2）

| 类型 | 修复方向 |
|------|----------|
| 格式问题 | 引号配对、段落标记 |
| 视角一致性 | 统一叙事者视角 |
| 年龄/时间暗示 | 补充隐性信息 |
| 标点规范 | 统一中英文标点 |

## 🤖 AI 味道识别清单

AI 写小说最常见的痕迹词和句式，审校时必须逐行排查：

### 痕迹词列表

| AI 痕迹词 | 人类写法 | 示例 |
|-----------|----------|------|
| 不禁 / 不由得 | 直接写动作 | ~~"他不禁叹了口气"~~ → "他叹气。" |
| 仿佛 / 如同 / 好像 | 直接描述或换比喻 | ~~"仿佛被雷劈中"~~ → "整个人一僵。" |
| 心中涌起 | 用身体感受代替 | ~~"心中涌起一阵寒意"~~ → "后背发凉。" |
| 深吸一口气 / 倒吸一口凉气 | 直接写反应 | ~~"她深吸一口气"~~ → "她吸气。" |
| 喃喃道 / 低语 | 直接引语 | ~~"他喃喃道：..."~~ → "他说。声音很轻。" |
| 嘴角勾起一抹... | 直接写表情 | ~~"嘴角勾起一抹苦笑"~~ → "他笑了一下。笑得苦。" |
| 眼中闪过... | 用动作代替 | ~~"眼中闪过一丝恐惧"~~ → "瞳孔缩了一下。" |
| 那一刻 / 那一瞬间 | 去掉，直接接动作 | ~~"那一刻他明白了"~~ → "他明白了。" |
| 不知过了多久 | 写具体感受 | ~~"不知过了多久"~~ → "脚麻了。" |
| 这就是... | 避免总结性结尾 | ~~"这就是命运"~~ → 停在动作或对话上 |
| 从此以后 / 从那以后 | 直接接后续 | 删掉，直接写"后来..." |
| 不由得 / 情不自禁 | 直接写动作 | ~~"不由得笑出声"~~ → "他笑出声。" |
| 似乎 / 大概 / 也许 | 删掉，直接说 | ~~"他似乎犹豫了一下"~~ → "他犹豫了一下。" |
| 一种...的... | 拆成短句 | ~~"一种说不清的悲伤涌上心头"~~ → "心里难受。说不上来为什么。" |
| 渐渐地 / 慢慢地 / 缓缓地 | 换成动词 | ~~"他缓缓站起身"~~ → "他站起来。慢。" |
| 紧紧地 / 死死地 | 换成具体动作 | ~~"紧紧抓住"~~ → "攥住。指甲掐进肉里。" |
| 突然 | 去掉"突然"，动作本身就突然 | ~~"突然一声巨响"~~ → "一声巨响。" |
| 竟然 / 居然 | 去掉，读者自己感受 | ~~"他竟然还活着"~~ → "他没死。" |

### 句式痕迹

| AI 句型 | 问题 | 修复 |
|---------|------|------|
| 形容词堆砌 | "他英俊潇洒高大挺拔玉树临风" | 挑一个最关键的，或者用行动代替 |
| 比喻+情感+动作三段式 | 每句都要比喻+心理描写+动作 | 删掉心理描写，让动作说话 |
| 排比句式 | "不是...不是...也不是...而是..." | 保留一两个，其余改陈述 |
| 升华式结尾 | "这个故事告诉我们..." | 停在具体画面或对话上 |
| 上帝视角解释 | 跳出角色解释心理 | 用动作/对话暗示，不解释 |

## ✂️ 长句拆分规则

网文读者习惯短句。AI 最爱写长句，必须拆：

### 拆分阈值

| 指标 | 阈值 |
|------|------|
| 单句最长字符 | ≤ 15 字（含标点） |
| 15字以上句子占比 | ≤ 30% |
| 逗号间隔最长 | ≤ 12 字 |
| 句号前最长 | ≤ 20 字 |

### 拆分方法

**方法 1：句号切断**
```
原句: 他看着远方的夕阳，金色的光芒洒在他的脸上，让他想起了多年前和虞姬一起看日出的日子。
拆成: 他看夕阳。金色光洒在脸上。他想起虞姬。多年前，也是这个时候，也是两个人。
```

**方法 2：动词断开**
```
原句: 他猛地站起身拔出腰间的长剑指向敌人的咽喉大声吼叫着要决一死战。
拆成: 他站起来。拔剑。剑尖指着敌人喉咙。他吼了一声。"决一死战。"
```

**方法 3：逗号换句号**
```
原句: 江水是浑浊的，泛着泥沙的颜色，像一条疲惫的黄龙，缓缓向东流去。
拆成: 江水浑浊。泛着泥沙。像一条黄龙。黄色的。往东流。慢。
```

**方法 4：删除连接词**
```
原句: 虽然他已经很累了，但是还是咬着牙撑了起来。
拆成: 他累。咬牙。撑起来。
```

### 检查方法

每一轮迭代后，执行以下检查：
```python
# 统计 15 字以上句子占比
sentences = text.split("。")
long_sentences = [s for s in sentences if len(s) > 15]
ratio = len(long_sentences) / len(sentences)
if ratio > 0.30:
    print(f"长句过多: {ratio:.0%}，需要拆分")
```

## 📊 评分体系 (6 维度 × 10 分 = 60 满分)

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 结构完整性 | 10% | 段落长度均匀、无缺失、顺序合理 |
| 历史准确性 | 10% | 时间线、人物、事件、数字正确 |
| 张力与节奏 | 15% | 高潮密度、悬念设置、节奏变化 |
| 语言精炼度 | 20% | 比喻密度、词频、冗余度 |
| 平台适配度 | 15% | 开头钩子、高潮密度、字数达标 |
| 情感连贯性 | 30% | 人物情感线、意象呼应、伏笔回收 |

**换算**: 总分 / 60 × 100 = 百分制分数

## ✅ 每轮迭代验证清单

每次修复后，必须执行以下验证：

```
1. 段落顺序完整性: 01 → 02 → ... → N (无跳段)
2. 段落长度均匀性: 最短段 ≥ 100 字，最长段 ≤ 1200 字
3. P0 问题清零验证: 逐个检查上一轮 P0 项
4. P1 阈值验证: 比喻≤40, "不是"≤20, 关键词≤6
5. 引号配对: 左引号数 = 右引号数
6. 重复短语检查: 同一短语连续重复 ≤ 2 处
7. 关键意象唯一性: 核心意象（如"抱不住"）只出现 1 次
8. 字数达标: 总字数 ≥ 平台最低要求（如番茄 ≥8000）
9. AI 痕迹词清理: 18 个常见 AI 痕迹词全部清零或 <3 处
10. 长句比例: 15 字以上句子占比 ≤ 30%
```

## ⚠️ 注意事项

1. **原稿永远不动**: 所有修改在 `story_vN.md` 上
2. **每轮一个文件**: V2 → V3 → V4，不覆盖上一版本
3. **系统化处理**: 不要跳过任何 P0 问题
4. **验证先行**: 修复前先验证问题存在，修复后再验证已清除
5. **保留风格**: 短句重复等刻意写作手法不要误杀
6. **自动循环**: 不要等用户确认每一轮，自动执行直到收敛
7. **最终汇报**: 收敛后给出完整版本历史对比表
